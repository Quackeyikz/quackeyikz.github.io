Laravel Cheatsheet

Code and commands cheatsheet for Laravel Framework

# Installation

Composer


                    composer create-project laravel/laravel:^11.0 project_name

Create a new Laravel projet with specified version using Composer.

Getting Started


                    php artisan serve

Run the Laravel project locally, execute the command while in the root directory.

# Migration

Table Creation


                    php artisan make:migration create_names_table

                    php artisan make:migration create_names_table --create=names

Creating a new file in the 'database/migrations' folder. Contains functions to create (up) and delete (down) relation.

Migration


                    php artisan migrate

                    php artisan migrate:rollback
                    
                    php artisan migrate:refresh

                    php artisan migrate:fresh

Running every migration file, will create table if not exist. Rollback to undo migration, fresh to start with empty table (fresh).

DDL & DML


                    $table->string('name');
                    $table->enum('col_name', ['choice1', 'choice2'])->default('choice1')->change();
                    $table->integer('salary')->nullable();
                    $table->unsignedBigInteger('company_id')->after('salary');
                    
                    $table->foreign('company_id')->on('companies')->references('id')->onDelete('cascade');

Defined in a up() function to determine definition, and manipulation. Use change() function to do an update from specified table. Recommended to do it in a different migration file.

# Model

Model


                    php artisan make:model MyModel

Eloquent


                    protected $table = 'employee';

                    protected $fillable = [
                        'name',
                        'date',
                        'address'
                    ];

Query could be written in the controller, retreiving data ONLY on what's written in the $fillable variable inside the model. Also, if Laravel mistook the table name (tends to assume the table name is plural), specify using $table variable.
